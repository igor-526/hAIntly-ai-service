# HAIntly AI Service (`ai-service`)

Сервис AI-оценки вакансий и генерации сопроводительных писем.

> Статус: архитектурная заготовка. Модели, prompts и контракты будут определены отдельными OpenSpec changes.

## Ответственность

- оценка релевантности вакансии выбранному резюме от 1 до 100;
- генерация сопроводительного письма под конкретную вакансию;
- управление AI-моделями и промптами;
- хранение статистики потребления токенов;
- нормализация ошибок внешнего AI-провайдера.

## Технологии

FastAPI, PostgreSQL, Celery и OpenAI-совместимый клиент для OpenRouter.

## Интеграции

- HTTP ← `main-be` для AI-настроек;
- NATS JetStream ↔ `vacancy-service` для заданий и результатов;
- HTTPS → OpenRouter.

## Границы и безопасность

- Сервис не управляет вакансиями и пользовательскими статусами.
- Провайдер подключается через адаптер, а доменная логика не зависит от SDK.
- API keys и полные prompts с персональными данными не логируются.
- AI score не является гарантированной истиной; контракт должен позволять отобразить статус и ошибки обработки.
- Live AI-вызовы не входят в обычный unit test suite.

Будущая документация должна добавить модель задач, provider configuration, env, token accounting, команды и тестовые режимы.
